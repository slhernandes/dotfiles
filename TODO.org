Mutating a plain JS array that's already assigned as a Repeater.model generally doesn't notify the view —
you usually need to reassign the property for QML to pick up the change. If window icons ever seem to only
update when you switch workspaces (rather than immediately on open/close), this is why. Worth verifying it
behaves as expected once you're on a dogfoodable setup.
